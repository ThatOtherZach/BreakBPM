# Replit Agent Prompt: Implement "Lucky Break Pass" for BreakBPM

You are an expert full-stack developer helping me build on my existing BreakBPM project (a billiards/pool scoring and stats tracking web app running on Replit). The app already has game modes (starting with 8-ball), shot logging, stats tracking (including BPM - balls per minute), user accounts, and game history.

## Vision & Goal

Implement **Lucky Break Pass** as the primary (and initially the only) way for users to unlock paid access/features in the app.

**Core mechanic**:
- User pays a one-time **$5.99 USD** (in crypto, preferably USDC or other stables via Coinbase Commerce).
- They are **guaranteed at least a Monthly pass**.
- A server-side RNG (driven purely by global/collective shot data across all users) then decides if they get upgraded all the way to a **Lifetime pass** instead.
- This replaces traditional recurring subscriptions (which testing showed reduces enthusiasm).
- **Lifetime holders get a powerful perk**: They can generate a free redeemable **Day Pass** (24h access) every 12 hours. This can be used by the lifetime holder themselves or shared/gifted to friends or new players. This turns big winners into ongoing "pass distributors" and makes the app very welcoming to newbies.
- The whole system is intentionally generous — we're almost giving away access — but the gamble + shared global data + lifetime sharing mechanic makes it fun, viral, and sustainable.

**Name to use**: "Lucky Break Pass" (or "Luck Break Pass" if you prefer the shorter version).

### 2. RNG & Global Shot Data Logic (The Heart of the Feature)

**Important shift**: We are **not** using individual user shot logs for personalization or bias. Instead, the RNG is driven entirely by the **global/collective shot data** across *all* users. This creates a living, shared randomness pool that grows every time anyone (including you) logs shots and games. New and existing players all contribute to and benefit from the same evolving dataset.

**Base odds** (make these easily configurable in one place, e.g. a constants file or admin setting). Suggested starting point (adjust as needed):
- Monthly pass: **90%**
- Lifetime pass: **10%**

(The removal of the Yearly tier keeps the reveal cleaner and more exciting — you either get the safe monthly floor or you hit the big Lifetime win.)

**How the global data drives the RNG**:
- Maintain a rolling/global stats object or recent activity buffer that aggregates meaningful signals from *all* shot logs in the system. Examples of useful aggregates:
  - Total shots logged across the entire user base (running total)
  - Recent shot volume / activity level (last 1h, 24h, 7d)
  - Global average pot rate / accuracy
  - Distribution or patterns in ball numbers, break outcomes, BPM spikes, etc.
  - Any other derived metrics that already exist or are easy to compute from the shot logging system.
- On each Lucky Break purchase, the RNG samples from (or is seeded by) this global dataset + the payment transaction hash. 
- The exact sampling method is flexible — you can propose a clean implementation (hashing recent global records, using aggregate stats as entropy/weighting factors, weighted random based on current global "energy", etc.). The goal is that the odds feel influenced by the living, collective activity of the entire BreakBPM community rather than pure server randomness or any single user's data.
- As more people play and log shots, the global dataset grows and evolves, making future rolls feel connected to the community's momentum.

**Flavor text on reveal**:
- Reference the *global/collective* energy instead of personal stats. Examples:
  - "The global break community just logged 12,847 shots this week — the cue gods are feeling generous..."
  - "With the recent surge in 8-ball activity across the app, your roll caught the wave..."
  - "The pool is alive tonight. Lifetime unlocked."

**Transparency & Trust**:
- On the result/reveal screen, show a "Roll verified" section that includes:
  - Transaction / charge ID from Coinbase
  - A seed or hash used for the roll (so advanced users can verify fairness if they want)
  - The final tier they received + the current base odds
  - A short note like "Seeded from global shot activity + tx hash"

Make the RNG deterministic given the same seed + global data snapshot so it can be reproduced for support/debugging. Store a snapshot of the relevant global stats at the time of each roll if needed for auditability.
